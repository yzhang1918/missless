# Harness Index

Status: Active

## Purpose

Provide one clear place to track and archive agent-harness workflow/process improvements separately from product delivery.

## Status Legend

- `Active`: currently used harness assets
- `Draft`: proposal/template not yet adopted
- `Superseded`: replaced harness asset

## First Stop

If you ask "what should we improve in the harness next?", start from GitHub Issues tagged for harness scope.

For a direct owner request, run discovery first and create a plan under [Active Plans Folder](./active/README.md) once the approach is approved.
