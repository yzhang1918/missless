# Execution Plans Index

Status: Active

## Purpose

Provide one clear place for product-delivery tactical tracking and product plan history.

## Status Legend

- `Active`: currently used planning assets
- `Draft`: template/proposal not yet adopted
- `Superseded`: replaced planning asset

## First Stop

If you ask "what should we do next?", start from GitHub Issues tagged for product scope.

For a direct owner request, run discovery first and create a plan under [Active Plans Folder](./active/README.md) once the approach is approved.
