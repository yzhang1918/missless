# Knowledge Workflow

Status: Active

## Main Flow

1. User submits a source via available interface.
2. System acquires and normalizes full source content.
3. System extracts atoms/artifacts from whole-content understanding.
4. System anchors evidence segments to extracted outputs.
5. System aligns outputs against existing knowledge.
6. System presents proposal with rationale and candidate evidence.
7. User reviews and controls persistence decisions.
8. System commits approved changes and updates run/event records.

## Review Actions

- Accept all or selected items
- Reject selected items
- Edit atom/artifact fields
- Override alignment decisions

## Commit Outcome

- Persist approved nodes and edges
- Persist event trail for replay and learning
- Update quality, overlap, and source-history signals
